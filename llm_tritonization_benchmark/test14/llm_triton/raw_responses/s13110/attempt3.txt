### Required Strategy: i-sequential, j-parallel

**Implementation Pattern (SINGLE KERNEL with in-kernel loop):**
- Python wrapper: launch ONE kernel with `grid = (triton.cdiv(j_size, BLOCK_SIZE),)`
- Triton kernel: use `for i in range(...)` loop INSIDE the kernel
- Triton kernel: parallelize `j` values using VECTORIZED operations within each block

**Why in-kernel loop is safe:**
- The sequential dimension `i` has dependencies that require ordering
- But within each `i` iteration, all `j` values are independent
- Reading from previous `i` iteration is safe because the in-kernel loop ensures ordering

**Example structure:**
```python

def kernel(...):
```
