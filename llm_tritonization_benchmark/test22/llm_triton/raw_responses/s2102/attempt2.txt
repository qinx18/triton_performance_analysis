**⚠️ CRITICAL: Identity Matrix / Diagonal Race Condition**

This code has TWO statements that can write to the same position:
1. `arr[j][i] = 0` - zeros all elements (parallel over j)
2. `arr[i][i] = 1` - sets diagonal (depends only on i)

**RACE CONDITION**: When j == i, both statements write to the same element!
If one block sets the diagonal while another block zeros the same position, you get wrong results.

**SOLUTION**: Each block should only set diagonal elements within its j range using MASKED stores:
```python
@triton.jit
def kernel(arr_ptr, N: tl.constexpr, BLOCK_SIZE: tl.constexpr):
    pid = tl.program_id(0)
    j_offsets = tl.arange(0, BLOCK_SIZE)
    j_idx = pid * BLOCK_SIZE + j_offsets
    j_mask = j_idx < N

    for i in range(N):  # Sequential loop
        # Zero all elements in this block's range
        ptrs = arr_ptr + j_idx * N + i
        tl.store(ptrs, 0.0, mask=j_mask)

        # Set diagonal ONLY if i is in this block's j range
        # Use MASKED store - each block only sets its own diagonals
        diag_mask = j_mask & (j_idx == i)
        diag_ptrs = arr_ptr + j_idx * N + j_idx  # diagonal positions
        tl.store(diag_ptrs, 1.0, mask=diag_mask)
```

**Key insight**: `diag_mask = j_mask & (j_idx == i)` ensures only ONE thread
in ONE block sets each diagonal element - the thread where j_idx equals i.
